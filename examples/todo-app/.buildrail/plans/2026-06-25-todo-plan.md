# 实现计划：待办事项（Todo）最小可用版

## 概述

把 5 个功能点拆成 4 个垂直切片任务，每个任务自包含实现+测试。按依赖顺序执行。

## 功能点映射

| 功能点 | 任务 |
|--------|------|
| 添加任务 | task-001 |
| 列出任务 | task-002 |
| 标记完成/删除 | task-003 |
| 持久化 | task-004（被 001-003 依赖） |

## 任务列表

---

### task-001: 实现持久化层（TodoStore）

**做什么：** 封装 JSON 文件的读写，支持 load/save，save 用原子写入（临时文件 + rename）。

**验收标准：**
- [ ] `TodoStore` 能从指定路径读取 JSON，文件不存在时返回空数组
- [ ] `save(tasks)` 写入后，重新 load 能拿到相同数据
- [ ] save 用临时文件 + rename 实现（避免写到一半崩溃损坏）
- [ ] `npm test` 通过

**涉及文件：**
- src/store.js（新增）
- src/store.test.js（新增）

**依赖：** 无
**预估大小：** S

---

### task-002: 实现 TodoService（增删查改逻辑）

**做什么：** 基于 TodoStore 实现 add/list/done/delete 四个操作，纯逻辑层不含 CLI。

**验收标准：**
- [ ] `add('买菜')` 后 list 返回含一条 `{title:'买菜', done:false}` 的记录
- [ ] `done(1)` 后，第 1 条记录 `done: true`
- [ ] `delete(1)` 后，list 不再含该记录
- [ ] 序号是 1-based，按数组顺序
- [ ] `npm test` 通过

**涉及文件：**
- src/todo.js（新增）
- src/todo.test.js（新增）

**依赖：** task-001
**预估大小：** M

---

### task-003: 实现 CLI 入口

**做什么：** 解析命令行参数，调用 TodoService，打印结果。支持 add/list/done/delete 子命令。

**验收标准：**
- [ ] `node bin/todo.js add 买菜` 后 JSON 文件出现一条记录
- [ ] `node bin/todo.js list` 打印带序号和 [ ]/[x] 标记的列表
- [ ] `node bin/todo.js done 1` 把第 1 条标记为完成
- [ ] `node bin/todo.js delete 1` 删除第 1 条
- [ ] 无效命令打印用法提示，退出码非 0

**涉及文件：**
- bin/todo.js（新增）
- package.json（修改，加 bin 字段）

**依赖：** task-002
**预估大小：** S

---

### task-004: 数据文件路径与初始化

**做什么：** 默认数据文件路径为 `~/.todo.json`（用户主目录），首次运行自动创建。支持 TODO_FILE 环境变量覆盖（测试用）。

**验收标准：**
- [ ] 首次运行（文件不存在）不报错，自动创建空文件
- [ ] 设置 `TODO_FILE=/tmp/test.json` 后，数据写到该路径
- [ ] `npm test` 持久化测试用 TODO_FILE 隔离，不污染真实环境

**涉及文件：**
- src/config.js（新增）
- src/todo.js（修改，用 config 里的路径）

**依赖：** task-002
**预估大小：** S

---

## 执行顺序建议

```
task-001 (Store) ──→ task-002 (Service) ──→ task-003 (CLI)
                                    ↘──→ task-004 (Config)
```

task-003 和 task-004 都依赖 task-002 但互不依赖，可任意顺序。
